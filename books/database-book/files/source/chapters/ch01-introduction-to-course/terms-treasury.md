| Term / Concept | Definition | Business Significance | Examples |
|---|---|---|---|
| **Applied Database Management** | The competency of designing and querying relational databases to answer realistic business questions. | Helps students move from knowing database terms to building useful systems that support decisions. | Designing tables and later writing SQL in the Grading Database. |
| **Bloom's Taxonomy** | A learning framework that moves from remembering and understanding toward applying, analyzing, evaluating, and creating. | Helps students see why the course asks them to do more than memorize definitions. | RAT questions check readiness; labs and reflections push toward analysis and evaluation. |
| **Business Meaning** | The organizational purpose behind technical work; the reason a query, table, report, or dashboard is worth building. | Keeps technical work connected to decisions, performance, and real organizational needs. | Writing the business question before writing a SQL query. |
| **Core Competencies** | The five long-term capabilities the book develops across the course. | Gives students a clear picture of the skills they are building for data-driven business roles. | Foundational information literacy, applied database management, data integrity and system quality, performance-oriented analysis, and strategic problem solving. |
| **Data Integrity and System Quality** | The competency of using modeling, structure, and rules to keep data accurate, consistent, and reliable. | Reduces errors and helps organizations trust reports, dashboards, and decisions. | Preventing duplicate records or inconsistent grade data in the Grading Database. |
| **Digital Companion** | The book's online set of working files, sample data, SQL scripts, database files, and lab resources. | Lets students inspect and practice with real materials instead of treating concepts as abstract ideas. | The GitHub repository that stores course files and supporting examples. |
| **Foundational Information Literacy** | The competency of explaining how data, databases, and information systems support organizational decision-making. | Gives students the language to discuss data systems clearly before building or analyzing them. | Explaining how data, technology, people, and processes support business performance. |
| **Grading Database** | The guided instructional case that uses students, courses, assignments, attendance, and grades to teach database logic. | Provides a familiar setting so students can focus on structure, queries, and reporting. | Tracking students, deliverables, scores, attendance, and final-grade calculations. |
| **Guided Practice** | Step-by-step work in a familiar case before students attempt more independent application. | Builds confidence and reduces cognitive load while students learn technical patterns. | Let's Build activities using the Grading Database. |
| **Instructional Arc** | The book's continuous progression from data to tables, relationships, queries, analytics, and decisions. | Shows how technical topics connect instead of appearing as isolated units. | Data -> Tables -> Relationships -> Queries -> Analytics -> Decisions. |
| **Performance-Oriented Analysis** | The competency of interpreting reports, metrics, and outputs in ways that connect technical work to business performance. | Helps students explain what a number means and why it matters for action. | Reading a Power BI dashboard or grade report and connecting it to performance questions. |
| **PetVax Veterinary Hospital Database** | The applied lab case that transfers course ideas to a veterinary clinic setting. | Tests whether students can recognize familiar database patterns in a new business context. | Owners, pets, appointments, treatments, invoices, and clinic operations. |
| **Query** | A precise request for data, often written in SQL, that answers a specific question. | Turns stored records into usable evidence for analysis and decisions. | Asking which appointments are missed most often or which deliverables are missing scores. |
| **Resilient Problem Solving** | The habit of treating errors, failed queries, and revisions as clues for learning. | Helps students debug technical work calmly and persist through realistic data problems. | Reading an error message, checking column names, and testing a smaller query. |
| **SQL** | Structured Query Language, the standard language used to ask questions of relational data. | Lets analysts retrieve, filter, join, and summarize data to support business decisions. | Writing queries against the Grading Database or PetVax tables in later chapters. |
| **Strategic Problem Solving** | The competency of combining technical reasoning and business context to evaluate options and recommend decisions. | Connects data skills to managerial judgment and evidence-based recommendations. | Choosing the stronger database design or explaining a misleading metric to leadership. |
| **Student Mindset** | The set of habits that helps students succeed in technical business learning. | Encourages steady, professional, detail-oriented learning rather than guessing or avoiding errors. | Curiosity, professionalism, engagement, attention to detail, and resilient problem solving. |
| **Systems Thinking** | The ability to see how data, structure, analysis, and decisions work together. | Helps students trace problems through the full system instead of blaming one tool or report. | Following a misleading dashboard number back to source data, tables, relationships, and queries. |
| **Tool Ecosystem** | The connected set of platforms used across the workflow from collection to communication. | Helps students choose tools by role instead of treating one software package as the strategy. | Google Forms for collection, Access for structure, SQL for querying, and Power BI for reports. |
| **Transfer Practice** | The higher-order learning step of applying a familiar structure in a new business context. | Prepares students for workplace situations where the domain changes but the logic remains. | Moving from the Grading Database to PetVax clinic data. |
