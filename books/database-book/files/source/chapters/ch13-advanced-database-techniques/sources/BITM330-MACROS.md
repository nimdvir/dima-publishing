UNDERSTANDING
THE PURPOSE OF A
MACRO (1 OF 2)

• Macro—series of  procedures that can be organized

to automate tasks or provide user interaction

• Standalone macro—database object created and
used independent of  other controls or objects

• Embedded macro—executes when an event attached

to a control or object occurs

• Event—user-initiated action

Understanding the Purpose of a Macro (2 of 2)

STANDALONE
MACROS

• Standalone macros are not associated with any

event or object in a database

• They are created and used independently of  other

controls or objects

• They are initiated by the user or a script

32

DATA MACROS

• Data macros are associated with a table and are

triggered by table events

• They are useful for ensuring data integrity and

accuracy

• Data macros can only be associated with table events

33

EVENT MACROS

• Event macros are associated with an event

such as a button click or form open

• They can be used to automate tasks that are

tied to a specific event or object

• Event macros can be embedded or

standalone

34

COMPARISON OF
MACROS

• Standalone macros are independent of  objects,
data macros are associated with tables, and
event macros are associated with events

• Standalone macros can be initiated by a user
or script, data macros are triggered by table
events, and event macros are triggered by
specific events

• Standalone macros are useful for general
automation, data macros are useful for
ensuring data integrity, and event macros are
useful for automating specific tasks tied to an
event
