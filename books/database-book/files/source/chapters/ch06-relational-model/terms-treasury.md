# Chapter 6 - Term Treasury

*Essential Vocabulary - understand the key terms used throughout this chapter.*

![Term Treasury section image](../../../../../.images/Ch0%20General/sections/resize/terms-sized.min.gif)

**Candidate Key**
A field or combination of fields that could uniquely identify each row in a table.

**Composite Key**
A key made from two or more fields that work together to identify a row uniquely.

**Data Redundancy**
The repeated storage of the same fact in multiple rows or places.

**Functional Dependency**
A relationship in which one attribute, or one set of attributes, determines another attribute.

**Junction Table**
A table that resolves a many-to-many relationship by storing one row for each valid pairing between related records.

**Natural Key**
A key built from real-world data that is already meaningful and unique enough to identify a row.

**Referential Integrity**
The rule that every non-null foreign key value must match an existing referenced key.

**Relational Model**
A framework for organizing data into related tables so facts are stored once and connected through keys.

**Schema**
The overall structure of a database, including its tables, fields, keys, and relationships.

**Surrogate Key**
An artificial identifier, often auto-generated, that is used as a table's primary key.
