## Consistency Checks

Apply these before assigning credit. Any failure indicates the student did not actually do the work.

1. **Row count parity.** `q07_PetVax_Report_Normalized` must return exactly 28 rows — the same as `PETVAX_FLAT`. Fewer rows = missing FK links; more rows = unintended Cartesian product.
2. **Charge parity.** `SUM(ActualCharge)` from `q07` must equal `SUM(ActualCharge)` from `PETVAX_FLAT` (= `1800`). If they differ, `VISIT_TREATMENT` was populated incorrectly.
3. **Quiz / artifact match.** The student's short-answer for Check 12 (`1800`) must match the total in their actual `q07` result. Mismatch suggests the quiz was answered without doing the build.
4. **Distinct counts.** Re-run `SELECT COUNT(*) FROM <table>` on each normalized table to confirm: `OWNER` 5, `PET` 7, `VET` 3, `TREATMENT` 6, `VISIT` 10, `VISIT_TREATMENT` 28. Any number off by exactly the original 28 = a missing `DISTINCT`.

## Notes for the AI Grader

- **How to read the file.** Open the `.accdb` in Access (or use a headless Access ODBC connection). Confirm table names and row counts first. Then open the Relationships window and visually verify each of the five RI lines. Finally, run `q07` and compare `COUNT(*)` and `SUM(ActualCharge)` to the consistency checks above.
- **Partial credit.** Award per-table credit: missing one normalized table forfeits that table's row in the rubric but not the others. Missing one relationship costs 2 points (out of the 10 allotted to relationships), not the full 10.
- **Append-pattern flexibility.** Students may use the GUI Append Wizard, dragged design queries, or hand-written SQL. Score on correctness of resulting rows, not on SQL style. The only banned shortcut is direct datasheet entry (copy-pasting columns from `PETVAX_FLAT`), which defeats the lesson.
- **Generic / faked submission signals.**
  - `q07` row count is not 28, or `SUM(ActualCharge)` is not 1800.
  - Quiz Check 12 answer does not match the student's own `q07` total.
  - Normalized tables contain owners/pets/vets that do not appear in `PETVAX_FLAT` (sign of pasted ChatGPT sample data).
  - No saved append queries, only manually entered rows — flag for human review.
- **Optional extensions.** Steps from the "Optional Extensions" section are bonus only. Do not deduct for missing them. Bonus cap: +5 points total (`q08_Visit_Totals` = 2, `STAFF_NOTES` table = 2, dirty-data writeup = 1).
