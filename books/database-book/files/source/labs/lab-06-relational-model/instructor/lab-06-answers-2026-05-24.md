## Reference SQL for the three saved queries

These are the SQL statements that exist in `lab-06-petvax-relational-ANSWER.accdb`. Student-submitted queries may differ in formatting and join order — Access rewrites them on save. Grade on result, not exact text.

### `qryAppointmentBillingDetail` — 9 rows expected

```sql
SELECT PET.PetName,
       PET.AnimalType,
       APPOINTMENT.AppointmentDate,
       APPOINTMENT.AppointmentStatus,
       APPOINTMENT_SERVICE.ServiceType,
       APPOINTMENT_SERVICE.PaymentAmount,
       SERVICE_TYPE.StandardFee,
       [PaymentAmount]-[StandardFee] AS PaymentDifference
FROM SERVICE_TYPE
INNER JOIN (APPOINTMENT
INNER JOIN (PET
INNER JOIN APPOINTMENT_SERVICE
       ON APPOINTMENT.AppointmentID = APPOINTMENT_SERVICE.AppointmentID)
       ON PET.PetID = APPOINTMENT.PetID)
       ON SERVICE_TYPE.ServiceType = APPOINTMENT_SERVICE.ServiceType
ORDER BY APPOINTMENT.AppointmentDate;
```

Expected rows with nonzero `PaymentDifference`:

| AppointmentID | Pet | Service | Paid | Standard | Difference |
| --- | --- | --- | ---: | ---: | ---: |
| 104 | Charlie | Checkup | $0 | $80 | **-$80** (no-show, never billed) |
| 108 | Buddy | Vaccination | $50 | $65 | **-$15** (discounted) |

### `qryOwnerPetSummary` — 6 rows expected

```sql
SELECT OWNER.OwnerID,
       OWNER.OwnerName,
       Count(PET_OWNER.PetID) AS PetCount
FROM OWNER
LEFT JOIN PET_OWNER
       ON OWNER.OwnerID = PET_OWNER.OwnerID
GROUP BY OWNER.OwnerID, OWNER.OwnerName
ORDER BY OWNER.OwnerName;
```

### `qryExpectedRequiredVaccinations` — 7 rows expected (Stage A)

```sql
SELECT PET.PetID,
       PET.PetName,
       PET.AnimalType,
       SERVICE_TYPE.ServiceType
FROM PET, SERVICE_TYPE
WHERE SERVICE_TYPE.RequiresVaccineTracking = 'Yes';
```

This is the cross-join "universe" of pet × required-service pairs. Every pet paired with `Vaccination` (the only required-tracking service). Used as the input to the next query.

### `qryMissingRequiredVaccinations` — 5 rows expected (Stage B, anti-join)

```sql
SELECT e.PetID,
       e.PetName,
       e.AnimalType,
       e.ServiceType
FROM qryExpectedRequiredVaccinations AS e
LEFT JOIN (APPOINTMENT
     INNER JOIN APPOINTMENT_SERVICE
            ON APPOINTMENT.AppointmentID = APPOINTMENT_SERVICE.AppointmentID)
       ON e.PetID = APPOINTMENT.PetID
      AND e.ServiceType = APPOINTMENT_SERVICE.ServiceType
WHERE APPOINTMENT_SERVICE.AppointmentServiceID IS NULL
ORDER BY e.PetName;
```

The two-query split is intentional: Access's Jet SQL engine sometimes refuses the equivalent single-query form (`FROM (PET, SERVICE_TYPE) LEFT JOIN (...) ON cond1 AND cond2`) with a `JOIN expression not supported` error. The staged version is reliable and easier for students to debug.

## Notes for the AI Grader

Common faked or incomplete submissions and how to spot them:

| Symptom | Likely cause | Deduct |
| --- | --- | --- |
| Relationships window is empty | Student skipped Step 3 entirely | 5 pts (Relationships) + likely 9 more (queries can't be graded reliably) |
| Relationship lines exist but show no infinity symbol | RI not enforced on Edit Relationships dialog | 1 pt per line |
| Cascade Delete is checked on any relationship | Student skipped the warning | 1 pt per relationship |
| `qryOwnerPetSummary` returns 5 rows instead of 6 | Used `INNER JOIN` instead of `LEFT JOIN` — Hannah disappears | 2 pts (criterion partially missed) |
| `qryMissingRequiredVaccinations` returns 0 rows | Used `INNER JOIN` instead of `LEFT JOIN`, or omitted `IS NULL` | 3 pts (query fundamentally wrong) |
| `qryMissingRequiredVaccinations` returns 7 rows | Did not anti-join against `APPOINTMENT_SERVICE` (returned the full expected universe) | 1 pt |
| `qryExpectedRequiredVaccinations` missing but final query returns 5 rows anyway | Student wrote the all-in-one form and it happened to work | 0 (full credit on result) |
| `qryExpectedRequiredVaccinations` returns 28 rows | Forgot the `WHERE RequiresVaccineTracking = 'Yes'` filter | 1 pt |
| Query exists with the right name but is empty / errors | Saved without running | Grade as missing |
| Relationship report shows the right lines but no infinity symbol | RI was not enforced | catches the same problem the Access file check would catch |
| Submitted `.accdb` is the unmodified starter | No relationships, no queries | 14 pts off (everything except Check 1–3 in the quiz) |

When a query's row count is right but the SQL Access generated differs in join order, **do not deduct**. Access rewrites SQL on save. Grade behavior, not text.

When the relationship report uses overlapping table boxes that hide fields, deduct the readability point only — students get the layout-rearrange instruction in Step 8.
